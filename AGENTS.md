# General rules
Before doing any action, think deeply and plan your work. Always try to leverage subagents by splitting your task in little subtasks that can be exedcuted by a subagent. If some of those tasks are independent call subagents in parallel.

After any task, consider to upgrade the tests and the documentation files.

# Context files
You have access to some context files that you will read during your task when needed:

## GOTCHAS.md
This file contains gotchas that other agents before you have encountered, so:
- read it when you have doubts, or get stuck 
- write at the end of your session a gotcha that you have encountered. Here be concise and write the minimum needed information to unlock the next agent

## ARCHITECTURE.md
This file contains the software architecture of the project you are working in

## DEPLOY.md
Explains how to deploy the app
